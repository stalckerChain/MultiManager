📑 ТЕХНИЧЕСКОЕ ЗАДАНИЕ И КОНТЕКСТ ПРОЕКТА: MultiManager (v1.0.0)1. Текущий статус проекта (MVP на 90%)Продукт представляет собой AI-Driven Web Automation Platform — комбайн для сбора дропхант-урожаев. Базовый функционал полностью реализован и стабильно запускается: создаются профили, на фронтенде Vue 3 работает управление окнами, синхронизация вкладок, кликов мыши (по кривым Безье) и ввода с клавиатуры. Все нереализованные ранее фичи (облако, сторонние миграторы) заморожены и перенесены в TODO. Текущая задача — интеграция готового Python-фреймворка автоматизации stAuto0, внедрение ИИ-агента (MCP), миграция существующего инстанса и стабилизация архитектуры.2. Обновленная схема БД SQLite (app.db в WAL-режиме)База данных better-sqlite3 является единственным источником правды. Выдерживает до 100k аккаунтов. Высокоинтенсивное логирование вынесено в текстовые файлы (pino), в базу пишутся только финальные статусы задач.Расширенная таблица profiles (Крипто-MVP)id (TEXT UUIDv4, PK) — Системный ключ профиля для API.number (INTEGER) — Порядковый номер аккаунта (например, 1, 2 для маски auto_001).name (TEXT) — Пользовательское имя (например, auto_001).proxy_id (INTEGER FK → proxies) — Ссылка на прокси (ON DELETE SET NULL).fingerprint_seed (TEXT) — Зерно для генерации шума Canvas/WebGL движком CloakBrowser.platform (TEXT: windows | macos | linux), user_agent, screen_resolution (TEXT).hardware_cores, hardware_memory (INTEGER).extensions, tags, notes (TEXT, JSON-массивы).status (TEXT: stopped | starting | running), pid (INTEGER).email (TEXT), email_password (TEXT, шифруется AES-256-GCM).Блок X (Twitter): twitter_username (NOT NULL), twitter_password (NOT NULL), twitter_auth_token (NULL), twitter_email (NULL).Блок Discord: discord_username (NOT NULL), discord_password (NOT NULL), discord_token (NULL), discord_email (NULL).Блок Web3: wallet_evm_address (TEXT), wallet_evm_private (TEXT, шифруется), wallet_sol_address (TEXT), wallet_sol_private (TEXT, шифруется), wallet_password (TEXT, шифруется, дефолт asdfj*KK).Новые таблицы планировщика задачtasks: id (UUIDv4), name (квест), script_name (для main.py), schedule_type (once | daily | weekly | manual | archive), cron_expression (TEXT), is_active (INTEGER).task_executions: id (PK AUTOINCREMENT), task_id (FK), profile_id (FK), status (success | failed | running), last_run_at (DATETIME), log_file_path (TEXT).3. Инфраструктурные модули бэкенда (Node.js / Electron)3.1. Кроссплатформенный Anti-Zombie контрольПри spawn ядра CloakBrowser его PID пишется в SQLite. Демон каждые 5 сек проверяет процессы через process.kill(pid, 0). При краше статус сбрасывается в stopped. Остановка идет через tree-kill: отправка SIGTERM ➔ ожидание 8 сек ➔ жесткий SIGKILL.3.2. Стратегия бэкапов (Hot Backup API)Резервируется строго один файл app.db при каждом холодном старте приложения. Папки кэша браузеров полностью игнорируются. Копирование идет на лету через встроенный в better-sqlite3 метод db.backup(), что исключает повреждение WAL-файлов. Реализована ротация Rolling Window: дампы в папке backups/ старше 7 дней (168 часов) удаляются автоматически по mtime.3.3. Разрешение конфликтов портов и АвторизацияElectron Main ищет свободный порт в диапазоне 3000–3100 через net.createServer() и форкает Core-движок Node.js с аргументами --port=N и случайно сгенерированным токеном --api-token=SECRET. Все запросы к API защищены middleware Authorization: Bearer SECRET. Фронтенд Vue 3 и WebSocket-клиент подключаются к динамическому порту с использованием Exponential Backoff (1с → 2с → 4с → 8с) для автореконнекта.4. Интеграция Python Движка Автоматизации (stAuto0)Концепция полностью изменена: Node.js бэкенд MultiManager полностью инкапсулирует управление браузером, расширениями и сетью, а Python освобождается от инфраструктурного кода и занимается чистой Web3-автоматизацией квестов.4.1. Модификация архитектуры файлов stAuto0Точка входа (main.py): Полностью сохраняет обратную совместимость с CLI-аргументами ручного запуска из консоли (--headless, --project=concrete,paragraph, --log-name=, диапазоны номеров 001-010 или конкретный auto_001).Замена источников конфигурации: Чтение статического файла config/accounts.py полностью удалено. При старте main.py парсит переданные диапазоны аккаунтов и делает локальный HTTP-запрос к MultiManager: GET /api/internal/profiles?range=001-010. Бэкенд возвращает JSON со всеми Web3-метриками, почтами и соцсетями из SQLite.Упразднение сетевого слоя в Python: Из фреймворка полностью удаляется модуль Core/proxy.py и проверка прокси через aiohttp. Вся сетевая валидация, ротация и прокси-чекеры происходят внутри Node.js до запуска браузера.Смена логики BaseBrowser (Core/browser.py): Метод launch() переписан. Теперь он отправляет HTTP-запрос POST /api/browser/{profile_id}/start к MultiManager. Бэкенд настраивает сеть, подмешивает фингерпринт, запускает CloakBrowser, самостоятельно осуществляет авто-авторизацию (разлок) в расширении кошелька Zerion по CDP с использованием wallet_password из SQLite, и возвращает в Python готовый ws_endpoint. Python-скрипт просто делает connect_over_cdp(ws_endpoint). Старый метод browser.login_zerion() в Python удален.4.2. Стек вызова проекта-плагина (Пример: Concrete)textmain.py ➔ run_account()
           ├─ HTTP POST /api/browser/{profile_id}/start ➔ Запуск и авто-логин Zerion силами Node.js
           ├─ BaseBrowser(ws_endpoint) ➔ Подключение по CDP (Playwright)
           └─ browser.run_project(ConcreteProject) ➔ Выполнение _process() квеста
Используйте код с осторожностью.5. Конвейер развертывания фермы (Wallet Factory)Обеспечивает параноидальную безопасность: сид-фразы полностью исключены из базы данных SQLite, RAM бэкенда и интерфейса GUI.Генерация (create_wallets.py): Запускается из консоли. Делает SQL-запрос SELECT MAX(number) FROM profiles для инкрементации индекса (например, продолжает с 16). На базе mnemonic и eth-account генерирует 24 слова BIP39, деривирует EVM и Solana адреса. Записывает в SQLite только публичные данные и дефолтный пароль asdfj*KK. Сами 24 слова пишет строго во временный локальный файл config/auto_sids.py.Автоматическое распределение почт: Перед генерацией скрипт читает config/free_email.txt. В цикле извлекает по одной почте и сразу пишет в поле email в SQLite. Если почт не хватает, создает профили с NULL и алертит. По завершении перезаписывает free_email.txt, удаляя использованные строки.Инициализация в браузере (init_wallet4browser.py): Запускается локально сразу после генерации. Читает сидки напрямую из временного файла config/auto_sids.py. Поочередно открывает CloakBrowser через API, заходит на страницу онбординга Zerion (chrome-extension://...) и пошагово вбивает 24 слова. Если видит селектор Session expired, пропускает аккаунт (идемпотентность). Поддерживает keep_open=true для ручной инспекции одного аккаунта.Уничтожение следов: После инициализации всей партии пользователь вручную удаляет файл config/auto_sids.py. В системе остаются чистые рабочие сессии Chromium без следов мнемоник на диске.6. Одноразовая консольная миграция (Console Only)Для переезда существующего «жирного» инстанса со старыми куками и сессиями GUI-интерфейс не пишется. Задача решается двумя изолированными консольными Python-скриптами:scripts/migrate_to_sqlite.py: Читает старый config/accounts.py, генерирует системные UUIDv4, распарсивает строки прокси host:port:user:pass, делает пакетную транзакцию в новую SQLite app.db и выгружает временную карту соответствия mapping.json ({"auto_001": "UUIDv4"}).scripts/migrate_profile_dirs.py: Читает mapping.json и физически копирует папки Chromium-сессий из старого каталога автоматизации в целевую системную папку %APPDATA%/CloakManager/profiles_data/{UUIDv4}/ через shutil.copytree, полностью сохраняя куки и авторизации кошельков. Скрипты удаляются после миграции.7. Слой ИИ-Агента и Генератор Скриптов (MCP)Встроенный Терминал: В GUI Vue 3 интегрируется консоль через xterm.js + node-pty на бэкенде для вывода логов автоматизации.Мультимодальный анализ: PicoAgent анализирует DOM-дерево по CDP + делает скриншоты страницы (Page.captureScreenshot), отправляя их в мультимодальную модель (GPT-4o / Claude 3.5) для прохождения капч и сложных интерфейсов.Режим обучения (Recorder): FastMCP-сервер (mcp_server/server.py) перехватывает CDP-события кликов и ввода пользователя. ИИ-модель агрегирует их и компилирует в готовый питоновский класс проекта, наследуемый от BaseProject, автоматически сохраняя его в папку projects/ для последующего запуска планировщиком MultiManager на любых аккаунтах.
